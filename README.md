# 《国运：三国招贤阁》MVP

一款基于Godot 4.x开发的三国题材挂机招募类游戏。

## 📁 目录结构
```
san-guo-recruit/
├─ project.godot          # Godot项目配置
├─ scenes/                # 场景文件
│  ├─ main.tscn           # 主界面
│  ├─ recruit.tscn        # 招募界面
│  ├─ lineup.tscn         # 阵容编辑界面
│  └─ bongo_cat.tscn      # Bongo Cat互动组件
├─ scripts/               # GDScript脚本
│  ├─ idle_manager.gd     # 挂机系统（单例）
│  ├─ hero_library.gd     # 武将库管理（单例）
│  ├─ recruit.gd          # 招募系统
│  ├─ lineup.gd           # 阵容编辑
│  ├─ bond_manager.gd     # 羁绊系统（单例）
│  ├─ battle_manager.gd   # 战斗系统（单例）
│  ├─ pvp_manager.gd      # 伪PVP系统（单例）
│  ├─ bongo_cat.gd        # Bongo Cat互动
│  ├─ main.gd             # 主界面逻辑
│  └─ save_manager.gd     # 存档系统（单例）
├─ data/                  # 数据文件
│  └─ heroes.json         # 武将数据（11个武将）
├─ assets/                # 美术资源（待添加）

   ├─ test_battle.gd      # 战斗系统测试
   └─ test_all.gd         # 全量测试入口
```

## 🎮 核心功能

### ✅ 已实现功能
1. **挂机系统**
   - 每秒自动产出国运点
   - 支持临时倍率（Bongo Cat提供）
   - 最多12小时离线收益
   - 接口：`get_current_gdp()`、`add_gdp()`、`set_temp_multiplier()`

2. **武将系统**
   - 11个预设武将（魏3、蜀3、吴3、群2）
   - 武将库管理：收集、碎片系统
   - 支持稀有度：白、绿、蓝、紫、橙

3. **招募系统**
   - 消耗100国运点招募一次
   - 概率分布：白70%、绿25%、蓝5%
   - 重复武将自动转化为碎片
   - 招募结果弹窗显示

4. **阵容系统**
   - 3x3九宫格布局，共9个位置
   - 点击选择武将放置
   - 自动检测重复武将
   - 不同势力显示不同颜色

5. **羁绊系统**
   - 魏国≥3人：攻击+10%
   - 蜀国≥3人：暴击率+15%
   - 吴国≥3人：技能触发概率+10%
   - 群雄≥3人：随机获得上述一种效果

6. **战斗系统**
   - 回合制战斗，按速度决定出手顺序
   - 普攻+技能系统（20%基础触发概率）
   - 伤害公式：`最终伤害 = 攻击 × 倍率 × (1+攻击加成) × (1-防御减伤)`
   - 完整战报记录
   - 羁绊效果自动应用

7. **伪PVP系统**
   - 每日生成15个AI对手，战力动态调整
   - AI阵容70%概率激活羁绊
   - 每日10次挑战机会
   - 胜利交换排名，获得威名奖励

8. **Bongo Cat互动**
   - 三种状态：idle、tap、fever
   - 按键触发1.5倍加速5秒
   - 每秒按5次触发3倍暴走10秒
   - 经验升级系统

9. **存档系统**
   - 每5分钟自动保存
   - 支持手动保存
   - 数据持久化：GDP、威名、武将库、阵容、排名等
   - 支持导出/导入存档

## 🎯 快速开始

### 环境要求
- Godot 4.x 版本

### 运行步骤
1. 克隆/下载项目到本地
2. 用Godot打开 `D:\UGit\san-guo-recruit\project.godot`
3. 按F5运行游戏
4. 初始赠送1000国运点，可以直接开始招募

## 🔧 开发说明

### 单例使用方式
所有全局管理器都可以直接通过单例访问：
```gdscript
# 获取当前国运点
var gdp = IdleManager.instance.get_current_gdp()

# 添加武将
HeroLibrary.instance.add_hero("wei_caocao")

# 开始战斗
var result = BattleManager.instance.start_battle(attacker_lineup, defender_lineup)
```

### 测试运行
1. 打开Godot编辑器
2. 运行测试脚本，执行自动化测试
3. 所有测试用例在 `tests/` 目录下
